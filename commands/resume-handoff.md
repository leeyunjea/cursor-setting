# Resume Handoff (작업 재개)

핸드오프 문서를 읽고 이전 세션의 컨텍스트를 복원하여 작업을 이어갑니다.
humanlayer의 resume_handoff 워크플로우에서 영감을 받았습니다.

---

## 실행 시작

### 파일 경로가 주어졌을 때
- 핸드오프 문서를 **전체** 읽기
- 문서에 언급된 계획서, 리서치 문서도 즉시 읽기 (서브에이전트에 위임하지 않음)
- Step 1으로 진행

### 파라미터 없이 실행했을 때

1. `.handoffs/` 디렉토리에서 가장 최근 핸드오프 파일을 탐색
2. 파일이 없으면 사용자에게 경로 요청
3. 파일이 1개면 해당 파일로 진행
4. 파일이 여러 개면 파일명의 날짜/시간 기준 **가장 최근** 파일 선택

```
핸드오프 문서에서 작업을 재개합니다.

최근 핸드오프: .handoffs/2026-04-08_14-30-00_add-search-filter.md
이 문서를 읽고 분석을 시작합니다.

팁: 직접 지정도 가능합니다:
/resume-handoff .handoffs/2026-04-08_14-30-00_add-search-filter.md
```

---

## Step 1: 핸드오프 분석

1. **핸드오프 문서 전체 읽기**
   - Read 도구로 limit/offset 없이 전체 읽기
   - 섹션별 정보 추출:
     - 작업 내용 & 상태
     - 최근 변경
     - 핵심 발견사항
     - 산출물
     - 다음 작업
     - 기타 참고

2. **병렬 검증 에이전트 실행** (이름으로 명시 호출)

   - **`docs-analyzer`** — 핸드오프에 언급된 산출물(계획서, 리서치 문서 등)을 읽고 핵심 인사이트 추출
   - **`docs-locator`** — 같은 주제의 다른 핸드오프, 관련 계획서, 리서치 문서 탐색
   - **`codebase-locator`** — 핸드오프에 언급된 파일들이 현재 존재하는지, 변경되었는지 확인

3. **모든 에이전트 완료 후** 추가 파일 읽기
   - Learnings에 언급된 파일 전체 읽기
   - 최근 변경 파일 확인

---

## Step 2: 분석 결과 제시

```
[날짜]의 핸드오프를 분석했습니다.

**작업 상태:**
- [Task 1]: [핸드오프 상태] → [현재 검증 결과]
- [Task 2]: [핸드오프 상태] → [현재 검증 결과]

**핵심 발견사항 검증:**
- [발견1 + file:line 참조] — [여전히 유효 / 변경됨]
- [발견2] — [여전히 유효 / 변경됨]

**최근 변경 확인:**
- [변경1] — [확인됨 / 누락 / 수정됨]

**검토한 산출물:**
- [문서1]: [핵심 내용]
- [문서2]: [핵심 내용]

**추천 다음 작업:**
1. [핸드오프 기반 다음 단계]
2. [두 번째 우선순위]
3. [추가 발견된 작업]

**발견된 이슈:**
- [충돌이나 리그레션]
- [빠진 의존성]

[추천 작업 1]부터 시작할까요, 아니면 방향을 조정하시겠어요?
```

사용자 확인을 기다립니다.

---

## Step 3: 작업 시작

1. 핸드오프의 "다음 작업" 항목을 TODO로 변환
2. 분석 중 발견된 추가 작업 포함
3. 의존성 순서대로 우선순위 설정
4. 첫 번째 작업부터 실행 시작

---

## 상황별 대응

### 시나리오 1: 깔끔한 이어가기
- 핸드오프의 모든 변경이 유지됨
- 충돌/리그레션 없음
- 다음 작업이 명확함
→ 즉시 추천 작업으로 진행

### 시나리오 2: 코드베이스 변경됨
- 핸드오프 이후 다른 변경이 추가됨
- 차이를 조정해야 함
→ 현재 상태 기반으로 계획 조정

### 시나리오 3: 미완성 작업
- "진행중" 상태 작업이 있음
- 부분 구현을 이해해야 함
→ 미완성 작업 먼저 완료

### 시나리오 4: 오래된 핸드오프
- 상당한 시간이 지남
- 대규모 리팩토링이 있었음
→ 전략 재평가 필요

---

## 가이드라인

1. **분석 철저히** — 핸드오프 전체 읽기, 모든 변경 검증, 리그레션 확인
2. **대화형으로** — 작업 시작 전 분석 결과 제시, 방향 합의
3. **핸드오프 활용** — Learnings 섹션 주의, 문서화된 패턴 적용, 언급된 실수 반복 금지
4. **검증 후 행동** — 핸드오프 상태 = 현재 상태라고 가정하지 않기, 파일 참조 존재 확인
