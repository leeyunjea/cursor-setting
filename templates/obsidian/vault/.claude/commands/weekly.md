# /weekly — 주간 회고 생성

이번 주 (또는 지정 주차) 주간 회고를 `10-Daily/weekly-{YYYY-Www}.md` 에 생성합니다.

## 프로세스

### Step 1: 대상 주차 결정

```bash
# 인자 없으면 이번 주
WEEK="${1:-$(date +%G-W%V)}"
```

### Step 2: 해당 주의 데일리 노트 수집

`10-Daily/` 에서 그 주(월~금)에 해당하는 노트 5개 읽기:
- 미존재 노트는 스킵 (휴가 등 가능)

### Step 3: 데일리 노트 분석

각 데일리 노트에서:
- **완료된 태스크** (`- [x]`)
- **미완료 태스크** (`- [ ]`) — 다음 주 이월 후보
- **"오늘 배운 것"** 섹션 — 일반화 가능한 항목 식별
- **미팅·인시던트 링크** — 키 이벤트 추출

### Step 4: 템플릿 적용

`_templates/weekly-review.md` 구조에 따라 작성:

**이번 주 한 일** — 완료 태스크 그룹화
**진행 중** — 데일리에서 반복 등장한 미완료
**막힌 것** — "막힘" "blocker" 키워드 검색
**배운 것** — "오늘 배운 것" 섹션 통합
- 일반화 가능 항목은 `30-Development/learning/` 으로 옮기는 것 제안
**미해결 / 이월** — 미완료 태스크 합치기
**다음 주 우선순위** — 빈 상태로 두고 사용자가 채우게

### Step 5: 후속 액션 제안

회고 작성 후:
- "30-Development/learning/ 으로 옮길 가치 있는 학습 N개 발견" → 사용자 확인
- 같은 주제가 반복 등장 → 별도 토픽 노트 생성 제안
- 미완료가 3주 연속 이월 → "재평가 또는 폐기 검토" 알림

## 출력 예시

```
✓ 10-Daily/weekly-2026-W17.md 생성 완료
  - 분석한 데일리: 5개 (월~금)
  - 완료 태스크: 12개
  - 이월 태스크: 4개
  - 학습 항목: 3개

후속 제안:
  📚 "idempotency key 설계" 학습 항목 — 30-Development/learning/ 로 분리 추천
  🔁 "리팩토링 PR 리뷰" 태스크 3주 연속 이월 — 재평가 필요
```
